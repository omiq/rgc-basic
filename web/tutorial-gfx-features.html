Graphics & browser features — quick reference

RGC-BASIC’s basic-gfx (native) and canvas WASM (web/canvas.html) share the same PETSCII screen, bitmap mode, sprites, and input model. Below is a compact map of newer commands; full programs live under examples/tutorial_gfx_*.bas.

Viewport scroll

Pan the visible framebuffer (text/bitmap + sprites) without moving your data in screen RAM. SCROLL dx, dy uses pixel offsets; SCROLLX() and SCROLLY() read the current values. 

SCROLL 16, 0     REM pan one character column (8 px) to the right
SCROLL 0, 0      REM reset
PRINT SCROLLX(); SCROLLY()

Runnable: examples/tutorial_gfx_scroll.bas

Virtual memory (POKE / PEEK)

Default bases match a C64-style map (screen $0400, colour $D800, etc.). Override with #OPTION memory … or the basic-gfx CLI -memory. 

#OPTION memory c64
REM or per region: #OPTION screen 1024

Runnable: examples/tutorial_gfx_memory.bas — see also README.md (Meta directives).

Sprites & tile sheets

LOADSPRITE slot, "file.png" loads a full image; add tw, th for a uniform tile grid. DRAWSPRITETILE draws one 1-based tile; SPRITEFRAME sets the default tile for DRAWSPRITE when you omit a crop. 

LOADSPRITE 0, "tiles.png", 8, 8
DRAWSPRITETILE 0, 100, 50, 2, 40   REM tile 2, z=40
SPRITEFRAME 0, 1
DRAWSPRITE 0, 180, 50, 40

Runnable: examples/tutorial_gfx_tilemap.bas + examples/tutorial_tile_sheet_demo.png

Gamepad

JOY(port, button) and JOYAXIS(port, axis) — native Raylib codes; in the browser, canvas.html maps them to the Standard Gamepad layout. 

PRINT JOY(0, 7); JOYAXIS(0, 0)

Short sample: examples/tutorial_gfx_gamepad.bas — continuous polling: examples/gfx_joy_demo.bas

How to run the examples

  • Native: make basic-gfx then ./basic-gfx examples/tutorial_gfx_scroll.bas
  • Browser: make basic-wasm-canvas, serve web/, open canvas.html, use Upload to VFS for a .bas file (and PNG assets in the same folder)
  • Index of demos: open examples/tutorial_gfx_index.bas in an editor — REM lines list every tutorial file
