RGC-BASIC Graphics 1.0

Release notes for the April 2026 graphics batch — a complete AMOS/STOS-class 2-D feature set landed in 36 commits between 2026-04-13 and 2026-04-18. Shape primitives, sprite stamping with rotation, an array-driven tilemap renderer, a 1bpp blitter with PNG/BMP I/O, a double-buffered VSYNC frame pipeline, named keyboard intrinsics, a global anti-alias toggle, and fourteen new example programs.

All commits sit in the Unreleased block of CHANGELOG.md (each entry explains what/why/implementation in full detail). This page is the short form with demo links.

Headline changes

Bitmap primitive set — full 2-D toolkit

Fourteen new drawing statements on top of the existing PSET / PRESET / LINE. Every outline has a matching solid fill; text now has a pixel-space stamp distinct from PRINT's character grid.

  • RECT x1,y1 TO x2,y2 / FILLRECT …
  • CIRCLE x,y,r / FILLCIRCLE x,y,r — midpoint disk
  • ELLIPSE x,y,rx,ry / FILLELLIPSE … — axis-aligned midpoint
  • TRIANGLE x1,y1,x2,y2,x3,y3 / FILLTRIANGLE … — scanline fill
  • POLYGON n, vx(), vy() / FILLPOLYGON … — N-sided from BASIC arrays, up to 256 vertices
  • FLOODFILL x, y — paint-bucket seed fill
  • DRAWTEXT x, y, text$ — pixel-space chargen stamp (no text-grid snap)

Sprite stamping + rotation + time-based animation

DRAWSPRITE has always tracked one persistent position per slot — fine for the player, useless for particles. SPRITE STAMP appends to a per-frame cell list, so N stamps of one slot in one frame all render at distinct positions. The sixth arg is rotation in degrees (raylib backend; canvas accepts but ignores). 

SPRITE STAMP slot, x, y [, frame [, z [, rot_deg]]]
SPRITE FRAME slot, ANIMFRAME(first, last, jiffies_per_frame)

ANIMFRAME(first, last, jiffies) returns a 1-based frame index cycling at the requested rate — no counter variable, no modulo maths. Feeds directly into SPRITE FRAME.

Tilemap renderer — one call, whole grid

DIM MAP(COLS*ROWS-1)
' fill MAP with 1-based tile indices (0 = transparent)
TILEMAP DRAW slot, x0, y0, cols, rows, MAP()

One interpreter dispatch stamps the entire grid. Runtime iterates in C (~1 µs yield overhead per tile on WASM matters — this batches it to one yield regardless of tile count). Negative x0/y0 shifts the map for smooth scrolling under a fixed viewport.

New metadata accessors: SHEET COLS(slot), SHEET ROWS, SHEET WIDTH, SHEET HEIGHT, plus SPRITE FRAMES(slot) and TILE COUNT(slot) for the same count under different intents.

Blitter Phase 1 — off-screen 1bpp surfaces with PNG/BMP I/O

  • IMAGE NEW slot, w, h / IMAGE FREE slot
  • IMAGE COPY src, sx, sy, sw, sh TO dst, dx, dy — clipped, overlap-safe
  • IMAGE LOAD slot, "path" — PNG/BMP/JPG/TGA/GIF → 1bpp (luminance threshold)
  • IMAGE GRAB slot, sx, sy, sw, sh — capture visible region into a fresh slot
  • IMAGE SAVE slot, "path.bmp" — export as 24-bit BMP

Slot 0 is the visible bitmap; copies work both directions without conversion. Scrolling / parallax / work-buffer patterns from AMOS/STOS-style BASIC now map 1:1. The SCREEN OFFSET / SCREEN ZONE / SCREEN SCROLL verbs originally proposed were dropped — IMAGE COPY with a user-managed offset covers both use cases without adding viewport state to the renderer.

Frame pipeline — VSYNC + double-buffered cells

raylib's GL framebuffer was already double-buffered at the pixel level, but the intermediate TILEMAP DRAW / SPRITE STAMP cell list was a single shared array. The compositor could snapshot mid-append and the user saw flicker. VSYNC closes the gap: 

CLS
TILEMAP DRAW ...
SPRITE STAMP ...
DRAWSPRITE ...
VSYNC           ' atomic commit + 1 display frame wait
GOTO loop

CLS clears the build buffer; the renderer reads the show buffer; VSYNC atomically flips build → show and yields one ~16 ms display frame (same timer-aware path SLEEP uses, so ON TIMER handlers keep firing).

Keyboard polling — named intrinsics

  • KEYDOWN(code) — 1 while held (lets A+D or A+W fire simultaneously for diagonal movement)
  • KEYUP(code) — inverse
  • KEYPRESS(code) — rising-edge latch, returns 1 exactly once per press (pause toggles, single-shot actions)

Replaces the PEEK(56320+ASC("W")) idiom. Survives #OPTION memory base changes. code = uppercase ASCII for letters/digits; specials: Up=145, Down=17, Left=157, Right=29, Esc=27, Space=32, Enter=13, Tab=9, Backspace=8.

Texture filter toggle — ANTIALIAS ON/OFF

Retro-vs-modern render choice. OFF (default) = nearest neighbour, hard pixels. ON = bilinear, smoothed. 

ANTIALIAS OFF   ' classic pixel art (default)
ANTIALIAS ON    ' smoothed scaling

Global setting — applies to every loaded sprite, the render target's upscale, and (once the Font system lands) DRAWTEXT. Also fixes a prior bug where sprite textures were hard-coded to BILINEAR even when the target was POINT (pixel art edges were slightly smudged).

Two-word command family + permanent concat aliases

New graphics verbs follow AMOS/STOS-style verb/noun grammar. Existing concat names stay forever — both spellings tokenise to the same opcode.

Two-word (canonical)Concat alias (permanent)
SPRITE LOAD slot, "file.png" [, tw, th]LOADSPRITE
SPRITE DRAW slot, x, y [, z [, sx, sy, sw, sh]]DRAWSPRITE
SPRITE FRAME slot, nSPRITEFRAME
SPRITE FREE slotUNLOADSPRITE
SPRITE FRAMES(slot) / TILE COUNT(slot)SPRITETILES(slot)
TILE DRAW slot, x, y, idxDRAWSPRITETILE
TILEMAP DRAW slot, x0, y0, cols, rows, map()DRAWTILEMAP

Pick per taste. Docs lead with two-word form for new material.

Parser fixes (shipped in the same cycle)

  • Single-line IF … THEN … ELSE … routes correctly in both directions (was half-broken)
  • IF (expr) MOD n = 0 THEN … — arithmetic continuation after (expr) now parses
  • TI/TI$ no longer swallows any identifier starting with "TI" (was eating TILE, TILEMAP, TIDE…)
  • GOTO from inside a THEN branch resets the inline-IF counter so the destination line's own ELSE isn't hijacked
  • SPRITEFRAME now invalidates the cached draw rect so animation actually advances (was locking to the first frame's crop)

IDE renderer default

The rgc-basic platform in the web IDE now defaults to the raylib-emscripten WebGL2 runtime — Graphics 1.0 features (SPRITE STAMP rotation, TILEMAP DRAW z-sort, VSYNC, ANIMFRAME) render the same way they do native. Set ?renderer=canvas on the IDE URL to force the old canvas path (useful for regression testing or on hosts without WebGL2).

The RGC BASIC Web ZIP download now bundles basic-raylib.js + basic-raylib.wasm instead of the canvas runtime for the same reason.
