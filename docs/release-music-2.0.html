RGC-BASIC 2.0 — Music, true colour, and a writable palette

Release notes for v2.0 / v2.0.1, tagged 2026-04-21. 55+ commits since v1.11.0, landing six big feature families on top of the Graphics 1.0 bitmap engine: streaming tracker music (MOD / XM / S3M / IT / OGG / MP3), a true-colour + indexed screen mode set (SCREEN 2 RGBA, SCREEN 3 256-colour indexed, SCREEN 4 640×400 RGBA), a writable 256-entry palette with hot rotation and .pal I/O, RGBA blitter surfaces with alpha compositing, IMAGE DRAW slot retargeting primitives into arbitrary-sized off-screen canvases, and SCROLL ZONE + SCROLL LINE for rectangle and per-scanline display-time scroll (demoscene copper- list effects without raster interrupts). Plus a polished language pass (string escape sequences, backslash integer divide, extended DRAWTEXT fg/bg/scale) and a --version flag on every binary, because you asked.

Full per-feature write-ups sit under the 2.0 / 2.0.1 sections of CHANGELOG.md (what / why / implementation for each entry). This page is the short form with demo links. The Graphics 1.0 release notes from April 2026 remain at release-graphics-1.0.html.

2.0.1 (fix pass)

Headline families

1. Music streaming — full transport + meters + metadata

One new slot pool (separate from the 1.11.0 LOADSOUND WAV slots). Raylib's raudio mixes streams concurrently, so BGM on music-slot 0 keeps playing while PLAYSOUND fires SFX. 

LOADMUSIC slot, "song.mod"          ' MOD / XM / S3M / IT / OGG / MP3
PLAYMUSIC slot                      ' start or restart
STOPMUSIC slot                      ' rewind + stop
PAUSEMUSIC slot / RESUMEMUSIC slot  ' suspend / continue
MUSICVOLUME slot, 0.0 .. 1.0
MUSICLOOP slot, 0 | 1
UNLOADMUSIC slot

Functions:

  • MUSICPLAYING(slot) — 1 while audible.
  • MUSICLENGTH(slot) — total track length in seconds. For MOD files, computed in gfx_sound.c via a pattern/order walk honouring Fxx (speed/BPM), Bxx (order jump), Dxx (pattern break) and E6x (pattern loop), with a 2-visit cap and 200 000-row budget to prevent runaway walks. Cached at LOADMUSIC. Non-MOD formats fall back to raylib's GetMusicTimeLength.
  • MUSICTIME(slot) — elapsed seconds via GetMusicTimePlayed. Wraps on loop.
  • MUSICPEAK() — 0..1 held-peak of the raudio master mix, decays ~0.92 per mix chunk. Drives VU meters.
  • MUSICTITLE$(slot) — 20-byte MOD title, trimmed.
  • MUSICSAMPLENAME$(slot, idx) — 22-byte sample name, idx 0..30.
  • MUSICCHANNELS(slot) / MUSICPATTERNS(slot) / MUSICORDERS(slot) / MUSICSAMPLECOUNT(slot) — parsed from the MOD tag + order table at load time.

Why a hand-rolled MOD length parser? Raylib's raudio calls jar_mod_max_samples inside LoadMusicStream — a full-song sample-by-sample simulation that blocks the main thread for 10+ seconds on Windows MinGW and under Emscripten ASYNCIFY. The 1.11.0 patch hollowed it out with frameCount = UINT_MAX, but that broke GetMusicTimePlayed (the (int)music.frameCount modulo underflows to -1). 2.0 fixes both: the hand-rolled parser computes real length in microseconds, and the patches/raudio_mod_skip_max_samples.patch hunk caps the GetMusicTimePlayed modulo at INT_MAX so MUSICTIME() advances honestly on MOD streams.

2. New screen modes — true colour + hi-res

The 1.9.x cycle shipped SCREEN 0 (text) and SCREEN 1 (1bpp bitmap, 16 palette indices). 2.0 adds three more:

  • SCREEN 2 new — 320×200 RGBA. Every pixel carries its own 32-bit RGBA, so gradients, semi-transparency, and full-colour PNG loads are native. New COLORRGB r, g, b [, a] and BACKGROUNDRGB pens; palette COLOR n still works and syncs the RGBA pen through the palette table. PRINT in SCREEN 2 stamps glyphs onto the RGBA bitmap with opaque paper so text composes cleanly over any background.
  • SCREEN 3 new — 320×200 8bpp palette-indexed. 64 KB per plane, reuses the SCREEN 1 colour plane. COLOR / BACKGROUND accept 0..255. Full 256-entry palette shared with SCREEN 1/2; PALETTESET / PALETTEROTATE visibly retint every drawn pixel on the next frame with zero redraw. Entries 0..15 are C64 defaults; 16..255 default to an HSV rainbow + greyscale band. Classic palette-cycling effects for water / fire / plasma / demo scrollers.
  • SCREEN 4 new — 640×400 RGBA. QB64-style desktop canvas, 1 MB per plane. Reuses every SCREEN 2 primitive with a wider clip; ideal for hi-res UI, maps, or printable title screens.

3. Writable 256-entry palette + hot rotation + .pal I/O

The C64 default 16-colour table is now the first slice of a 256-entry live palette that renders every frame. Every pixel written in SCREEN 1 or SCREEN 3 looks up its colour through this table, so rewriting an entry immediately retints every already-drawn pixel at that index.

  • PALETTESET i, r, g, b [, a] — set entry i (0..255) by RGBA.
  • PALETTESETHEX i, "#RRGGBB[AA]" — same but hex-string form; leading # optional.
  • PALETTERESET — restore C64 defaults (0..15) plus the HSV rainbow + greyscale default (16..255).
  • PALETTEROTATE first, last [, step] — shift entries [first..last] in place by step (default 1). One C-side memmove; renders visibly next frame without any bitmap redraw. Underpins copper-bar / raster-scroll demos.
  • PALETTE(i, chan) — read channel (0=R, 1=G, 2=B, 3=A).
  • PALETTEHEX$(i) — "#RRGGBB" (or "#RRGGBBAA" if alpha < 255).
  • PALETTELOAD "file.pal" / PALETTESAVE "file.pal" — plain-text i,r,g,b[,a] lines, one entry per line. Trivially diffable, editable in any text editor, interoperable with common pixel tools.

Note for 2.0: the earlier 0..15 cap on PALETTESET / PALETTESETHEX was a C64-era leftover. 2.0 lifts it to 0..255 so indices 16..255 can be retuned for copper-bar / demo rotation.

4. RGBA blitter surfaces + alpha compositing

Graphics 1.0's IMAGE NEW / IMAGE COPY etc. were 1bpp only. 2.0 adds full RGBA off-screen surfaces with a proper Porter-Duff source-over compositor:

  • IMAGE CREATE slot, w, h — allocate an RGBA off-screen surface (slots 1..31; 0 is the live framebuffer).
  • IMAGE LOAD slot, "file.png" now fills the RGBA buffer when the slot was pre-created via IMAGE CREATE, preserving the PNG's alpha instead of luminance-thresholding into a 1bpp mask.
  • IMAGE BLEND src, sx, sy, sw, sh TO dst, dx, dy — alpha-composited blit between RGBA slots. dst = 0 routes to the live SCREEN 2 / 4 framebuffer.

Existing IMAGE COPY, IMAGE SAVE, IMAGE GRAB all pick the right plane shape (1bpp or RGBA) from the slot. IMAGE SAVE auto-routes on extension: .png writes 32-bit RGBA (preserves alpha from a grab or from RGBA paint), anything else writes 24-bit BMP.

5. LOADSCREEN — PNG/BMP/JPG into any screen mode

One verb, four backends, chosen by the active SCREEN: 

LOADSCREEN path$ [, x [, y]]
  • SCREEN 0 (text) — cell-quantises to 1 fg + 1 bg hardware colour per cell + PETSCII block glyph. Image becomes ANSI art.
  • SCREEN 1 (1bpp) — Floyd-Steinberg dithers to the 16 hardware colours, stores per-pixel index in bitmap_color[].
  • SCREEN 2 / SCREEN 4 (RGBA) — copies PNG as-is, full colour + alpha preserved.
  • SCREEN 3 (indexed) — quantises to the 256-entry palette via nearest-RGB match; alpha < 128 maps to the current BACKGROUND index.

Pairs naturally with PALETTEROTATE in SCREEN 3 — load a PNG, rotate a slice of the palette, get animated water / fire / neon across the still image with no per-pixel redraw.

6. Extended DRAWTEXT — per-call fg, bg, font, scale

New 5-argument form on top of the 1.9 scale-only extension: 

DRAWTEXT x, y, text$                      ' defaults
DRAWTEXT x, y, text$, scale               ' legacy (1.9.7)
DRAWTEXT x, y, text$, fg, bg [, font [, scale]]
  • fg — palette index for glyph pixels. -1 keeps the current pen.
  • bg — palette index for the cell background. -1 = transparent paper (non-glyph pixels untouched). Paints the bounding rect before stamping.
  • font — parsed but currently ignored; reserved for LOADFONT when the Font system lands.
  • scale — 1..8 integer pixel-double, same as the legacy 4-arg form.

Makes reverse-video, chromed labels, and multi-pen HUD text a one-liner instead of a two-pass stamp. Pairs with SCREEN 2 / 4 for full-colour styling.

7. Language polish pass

  • String escape sequences — \n \r \t \0 \\ \" expanded at load time inside double-quoted strings. Runs in transform_basic_line before tokenisation so every screen mode and build (terminal, basic-gfx, WASM) picks them up uniformly.
  • \ integer divide — classic BASIC / QBasic floor-divide. (a \ b) * b + (a MOD b) == a for non-zero b. Pairs with the existing MOD operator.
  • Function string-parameter binding fix — string params in FUNCTION / DEF FN now bind under the stripped name, so FUNCTION F(A$) reads A$ inside the body correctly (was losing the $ in some paths).
  • OPEN / GET# / GETBYTE / PUTBYTE set the ST status variable on error instead of halting the program. Matches classic CBM BASIC behaviour: the program sees ST <> 0 on the next line and can branch.
  • --version / -v / -V on every binary — standard Unix version flag on basic, basic-gfx, and every basic-wasm* build. 
    $ ./basic --version
RGC-BASIC v2.0.1 (2026-04-21) — basic
https://github.com/omiq/rgc-basic
    Version string is injected at build time via git describe --tags --dirty --always; release date is the tagged commit's committer date. Useful for bug reports and checking which runtime the IDE is serving.

8. IMAGE DRAW slot — retarget primitives to an off-screen canvas

Mirror of the existing SCREEN DRAW n, but routes into the IMAGE pool instead of the fixed-size SCREEN plane pool: 

IMAGE CREATE 1, 1024, 64        ' arbitrary-size RGBA canvas
IMAGE DRAW 1                    ' retarget primitives here
LINE 0, 0 TO 1023, 63           ' draws into slot 1
DRAWTEXT 8, 8, "BAKED!", 2
IMAGE DRAW 0                    ' restore live framebuffer

Every SCREEN 2 / SCREEN 4 primitive (LINE, FILLRECT, CIRCLE, DRAWTEXT, PSET, POLYGON, …) reads bitmap_rgba + rgba_w/h; the retarget swaps that triple to point at the IMAGE slot, so every primitive lands in the off-screen canvas with no per-call dispatch. IMAGE DRAW 0 restores the saved triple. Unlocks: gradient-into-glyphs pre-baked scroller strips (finally — see demo-scroller-png.bas for the "pre-baked PNG" equivalent that used to be the only path), world-map assembly, HUD layer grabs, reusable sprite texture atlases built at runtime.

Non-RGBA screen modes (SCREEN 0 / 1 / 3) return a runtime error — primitives in those modes write different buffers and aren't reachable by the pointer swap. 1bpp canvases via IMAGE NEW similarly rejected.

9. SCROLL ZONE / SCROLL LINE — rectangle + per-scanline scroll

The display-time partner to IMAGE DRAW. Classic Amiga copper-list effects without raster interrupts.

Zones — named horizontal bands scroll independently:

SCROLL ZONE 1, 80, 40           ' declare zone id 1 = rows 80..119
SCROLL ZONE 1, -2               ' advance dx by -2 px this frame
                                '   (wraps on plane width, persistent)
SCROLL ZONE CLEAR 1             ' release the zone
SCROLL ZONE RESET 1             ' zero the offset, keep the rect

Up to 7 zones at once (ids 1..7). Zone state survives across frames — one advance call per frame = a smooth scroll, no bookkeeping. Parallax-multiple zones for sky / mid / ground bands. A zone at y=0..39 over the full screen = classic demo message-bar scroll with the rest of the screen static.

Per-scanline warp — each pixel row gets its own dx:

' Water ripple across rows 120..199
FOR Y = 120 TO 199
  SCROLL LINE Y, INT( SIN((Y + T) * 0.2) * 6 )
NEXT
VSYNC

Water, heat haze, flag wave, CRT jitter. Layer on top of a zone (the zone's bulk dx + the per-line wobble stack additively) for "scrolling text with ripple" effects.

SCROLL LINE RESET zeroes every row; SCROLL RESET nukes both zones + per-line state. Fast-path flag in the compositor means zero-cost pass-through when no scroll state is active.

10. Patched raylib built from source — native & WASM

2.0 drops the system-raylib path on every CI target. Homebrew, MinGW and apt all ship un-patched raylib — that's how the Windows MOD freeze slipped through for a cycle. The new build pipeline:

  • scripts/build-raylib-native.sh (POSIX) / scripts/build-raylib-native.cmd (Windows) clones raylib 5.5 into third_party/raylib-native/, applies patches/*.patch, and builds a static libraylib.a that basic-gfx links.
  • scripts/build-raylib-web.sh does the same for the Emscripten build under third_party/raylib-src/.
  • .gitattributes pins patches/*.patch text eol=lf so Git-for-Windows autocrlf doesn't silently rewrite them to CRLF and fail the git apply whitespace check.
  • Per-OS link libraries baked into the Makefile — macOS picks up -framework CoreAudio -framework AudioToolbox (without these miniaudio's InitAudioDevice SIGSEGVs), Windows gets -lole32 -loleaut32 -luuid for miniaudio's WASAPI backend plus static winpthread so the binary runs outside MSYS2.
  • CI workflows (.github/workflows/{ci,nightly}.yml) drop the old libraylib-dev / brew raylib / mingw-w64-raylib installs. "Not a full CI without the patched audio path" is now enforced end-to-end.

9. IDE quality-of-life

  • Asset auto-preload extended to DIM-held paths. The regex that fetches referenced assets into MEMFS used to only match verb-prefixed literals (LOADSPRITE 0, "ship.png"). 2.0 adds a pass that catches any string literal ending in a known asset extension, so programs that stash paths in variables or arrays work: 
    DIM PATH$(9)
PATH$(1) = "music/drozerix_ai_renaissance.mod"
LOADMUSIC 0, PATH$(1)           ' now auto-preloaded
    Covers mod / xm / s3m / it / ogg / mp3 / wav / png / pal / rgcfont / seq.
  • Autoplay-gate UX. Browsers keep every AudioContext suspended until a user gesture; previously that was invisible. 2.0 adds a pulsing speaker-off badge top-right while audio is muted, hidden the moment the first pointerdown / touchstart / keydown / click lands. Listeners registered on window, document, and the canvas directly, plus fullscreenchange, so clicks in fullscreen mode count as valid gestures too.
  • Pause button now silences music. The IDE's pause used to only flag the interpreter — the raylib render loop kept ticking gfx_music_tick so music never stopped. 2.0 suspends miniaudio's AudioContext on pause and resumes it on unpause, so paused programs are genuinely silent.
  • Sync script preserves music/ subdir. scripts/sync-rgc-basic.sh in the 8bitworkshop repo now copies tracker modules into presets/rgc-basic/music/ instead of flattening them alongside .bas files, so LOADMUSIC "music/foo.mod" resolves the same way in the IDE as natively.

New example programs

Demoscene scroller tutorial — five approaches

A progression from "zero assets, no engine tricks" to "full Amiga bitmap-font with sine wobble", so readers can pick the complexity level they want and learn the idea behind each approach.

demo-scroller-palette.bas Approach 1 — SCREEN 3 + PALETTEROTATE copper bars. Cheapest, no assets. Classic C64 raster-bar feel without raster interrupts. demo-scroller-composite.bas Approach 2 — per-frame RGBA gradient band + scrolling white text on top. SCREEN 2 + DOUBLEBUFFER. demo-scroller-png.bas Approach 3 — one long RGBA PNG strip (2400×40) with transparency, scrolled + sine-bounced as a sprite. Richest visuals for the lowest code. demo-scroller-sprites.bas Approach 4 — Amiga-style 32×32 bitmap font (scroller_font.png from ianhan/BitmapFonts), one SPRITE STAMP per letter with per-letter sine wobble. demo-scroller-reverse.bas Approach 5 — reverse-video gradient. Pre-bakes a 16×16 gradient tile into an RGBA IMAGE slot and stamps it per cell; DRAWTEXT punches black letter-shaped holes through.

Feature-specific demos

gfx_music_demo.bas Rewritten tracker demo. Keys 1..9 switch tracks. NOW PLAYING strip, MM:SS / MM:SS time, progress bar, 32-bar green/yellow/red VU meter driven by MUSICPEAK(), 32-bar volume meter. Exercises the full music API. gfx_palette_demo.bas Animated PALETTEROTATE — SPACE toggles pause. What's next (2.1+)

  1. Live MOD tracker info — MUSICROW(slot), MUSICORDER(slot), MUSICBPM(slot), MUSICSPEED(slot). Needs a small patch exposing jar_mod_context_t so BASIC can read the live per-frame position.
  2. Oscilloscope / spectrum — MUSICWAVE(i) returning the last N mix samples; keep the existing AttachAudioMixedProcessor callback, add a 512-sample ring buffer. Optional cheap FFT for spectrum bars.
  3. Per-channel meters — MUSICCHANVOL(slot, ch) / MUSICCHANPERIOD(slot, ch). Reaches into jar_mod's per-voice state.
  4. Multi-voice SFX — overlap two PLAYSOUND calls instead of the MVP single-voice rule. Deferred from rgc-sound-sample-spec.md section 3.4.
  5. LOADFONT / typography — wire the font_slot argument already parsed by DRAWTEXT to a real loadable font format.
